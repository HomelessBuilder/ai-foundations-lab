# Module 2 — Your First Project Folder
## Lab 2: "Give AI a Home"

*Status: Draft — for Diego's review.*

**What you'll walk away with:** A real project folder on your computer,
with a CLAUDE.md file inside it that you wrote — seeded from the
reflection you saved in Lab 1.

**Time:** About 60 minutes. Slower is fine.

---

### Before We Start

You've got a file sitting around called `what-i-want-to-build.txt`. Go
find it if you're not sure where it landed. We're about to give it a
real home — and give Claude Code something it's never had with you
before: a project to belong to.

---

### Step 1 — Make a Folder

Create a new folder on your computer. Name it something that makes
sense to you — `my-first-project` is fine if nothing comes to mind.
Wherever you keep your regular files is the right place. Nothing special
required.

> **Worth knowing:** to Claude Code, a folder isn't just storage — it's
> a boundary. Whatever folder you open it in becomes "the project" as
> far as it's concerned. That's why we're making a fresh one now instead
> of just working wherever `hello.txt` happened to land. A clean folder
> means a clean, defined project.

Move your `what-i-want-to-build.txt` file into this new folder.

---

### Step 2 — Open Claude Code Here

Open a terminal, navigate into your new folder using the trick from Lab
0's Step 5, and type `claude` — same as Lab 0, just from inside this
folder instead of wherever you were before.

> **This part trips people up sometimes.** If Claude Code doesn't seem
> to know about your folder or the file inside it, double check you
> actually opened the terminal *in* that folder before typing `claude`.
> Location is the whole trick here — close it, navigate in properly, and
> try again. Not a sign you did something wrong, just a "where am I"
> mix-up that happens to everyone at first.

---

### Step 3 — Write Your First CLAUDE.md

A `CLAUDE.md` is a plain text file that sits in a project folder and
tells Claude Code what it needs to know about this project — before you
say a single word in conversation. Think of it as context you write down
once, instead of context you'd otherwise have to repeat every time you
open a new session.

Let's build yours together with Claude Code, using what you already
wrote in Lab 1. Type something like:

> "There's a file in this folder called what-i-want-to-build.txt — can
> you read it, and help me turn it into a CLAUDE.md for this project?"

Claude Code will likely draft something back to you. Read it. Change
anything that doesn't sound like you, cut anything that doesn't feel
true, add anything missing. This has to end up being *yours* — not
because Claude Code did it wrong, but because a CLAUDE.md only works if
it actually reflects your project and your situation.

If you want a sense of what a simple one looks like before you start,
here's a bare-bones example:

```markdown
# My Project

## What This Is
I'm learning to build small things with AI help. Right now I'm
just getting started — no big project yet, just exploring.

## About Me
New to coding and terminals. Explain things in plain language.
I learn best by doing, not by reading long explanations first.

## What I Want Help With
Right now: understanding what AI can actually do for me day to
day, and getting comfortable enough to try small real projects.
```

Yours doesn't need to look exactly like this. It needs to sound like you.

---

### Step 4 — Prove It Persists

Here's the difference between a CLAUDE.md and just talking to Claude
Code in conversation, and it matters more than it might seem right now:

**Conversation** is for right now — a specific question, a specific ask,
something true only in this moment. **CLAUDE.md** is for things that
stay true across every session — who you are, what the project is, how
you like things explained. Conversation gets forgotten when the session
ends. CLAUDE.md doesn't.

Don't just take that on faith — prove it to yourself:

1. Close Claude Code entirely.
2. Open it again, in this same folder.
3. Ask it something like "what do you know about me?"

Notice that it still knows — not because it remembers you personally,
but because it reads the CLAUDE.md fresh every time. That's the whole
trick behind everything this course teaches: write it down once, and it
holds.

> **Worth knowing:** if it comes back sounding like it forgot
> everything, that almost always means the terminal reopened somewhere
> other than your project folder. Same "where am I" issue as Step 2 —
> close it, get back into the right folder, and try again.

### Where We Land

You just did something a lot of people who use AI tools never think to
do: you gave it a real, permanent piece of context instead of re-explaining
yourself every single time. That folder, and that file, are yours now —
they're the foundation everything else in this course gets built on top
of.
