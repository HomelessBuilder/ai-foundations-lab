# Module 7 — What Comes Next
## Lab 7: "Where This Goes"

*Status: Draft — for Diego's review.*

**What you'll walk away with:** A one-page "what's next" note, in your
own words, filed as the final entry in your journal.

**Time:** About 60 minutes. Slower is fine — there's nowhere left to
rush to.

---

### Before We Start

There's nothing new to learn today. This lab is about looking at what
you've actually built and deciding, in your own words, where you want to
take it from here.

Take a second before we start. A little while ago, you didn't have
Claude Code installed. Now you have a real project folder — with a
CLAUDE.md, a journal with your own words in it, a working micro-project,
and a git history protecting all of it. That's not a metaphor for
learning. That's the actual thing.

---

### Step 1 — Look Back

Ask Claude Code:

```
Can you show me everything that's in my project folder now?
```

Look at the list. Every single thing on it — the CLAUDE.md, the journal
entries, the handoff note, your micro-project, the git history — you
built. Not "watched get built." Built.

> **Worth knowing:** this is the same "show me what's here" move from
> Lab 4's closing step, and the same cold-reentry move from Lab 5. Same
> pattern, one more time — because that's what this whole course has
> been: a small number of moves, practiced until they're yours.

---

### Step 2 — A Light Look Ahead

Everything you've done so far has been deliberately small and safe.
There's more out there whenever you're ready for it — you don't need any
of it today, but it's worth knowing it exists:

- **Bigger projects** — everything you did today scales up. The same
  loop — context, build, document, re-enter — works on something with
  ten files just as well as it worked on one.
- **More of git** — `init`, `add`, and `commit` are the foundation.
  There's more underneath (going back to old snapshots, working with
  other people on the same project) for whenever you need it.
- **Sharper prompting** — the vague-vs-specific instinct from Lab 4 gets
  more powerful the more you practice it. That's a skill that just keeps
  paying off.

None of this is homework. It's just the horizon — so "what comes next"
in a minute doesn't feel like a blank page.

---

### Step 3 — Write Your "What's Next" Note

Have a real conversation with Claude Code about what you'd actually like
to try next, now that you have this set up. If you're not sure where to
start, try:

> "Now that I have this set up, here's what I'm curious to try next..."

When you've said what you actually think, ask it to save it:

```
Can you save this as the next numbered entry in my journal — a short note on what I want to try next?
```

Two or three things is plenty. This doesn't need to be a plan. It just
needs to be true.

---

We've got this. Go build something.
