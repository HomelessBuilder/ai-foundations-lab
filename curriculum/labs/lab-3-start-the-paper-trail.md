# Module 3 — The Learning Journal
## Lab 3: "Start the Paper Trail"

*Status: Draft — for Diego's review.*

**What you'll walk away with:** A journal folder living alongside your
project, with one real entry in it documenting what you've done so far.

**Time:** About 60 minutes. Slower is fine.

---

### Before We Start

You've got a project folder now, with a CLAUDE.md that tells Claude Code
who you are and what you're trying to do. That file answers "what's
true about this project." Today we're adding something different: a
place that answers "what actually happened."

That's the learning journal — and it's the piece that makes everything
else in this course actually stick.

---

### Step 1 — Make a Journal Folder

Inside your project folder — the same one with your CLAUDE.md — create
a new folder called `journal`.

> **Worth knowing:** CLAUDE.md and the journal do two different jobs,
> even though they're both just text files. CLAUDE.md holds what stays
> true — it barely changes once it's written. The journal holds what
> happened — it grows every time you use it. One is your project's
> identity. The other is its history.

---

### Step 2 — Write Your First Entry

Have a quick conversation with Claude Code about what you've actually
done so far — installing it, saying hello, building your CLAUDE.md.
Nothing fancy, just a real recap in your own words. If you're not sure
how to start, try:

> "Here's what I've done in this course so far..."

Then ask it to write that down:

```
Can you save a journal entry for what we just talked about, as journal/entry-1.md?
```

Claude Code will create the entry inside your journal folder — you'll
see it appear there.

If you want a sense of what a simple entry looks like, here's a
bare-bones example:

```markdown
# Entry 1 — Getting Started

## What Happened
Installed Claude Code, said hello to it for the first time, and
set up my project folder with a CLAUDE.md.

## What I Learned
Claude Code is different from a chat tool — it can actually create
files on my computer. Writing things down in CLAUDE.md means I
don't have to re-explain myself every session.

## What's Next
Starting to build something small in the next lab.
```

The `#` symbols just mark headings in a plain text file — don't worry
about them, Claude Code handles it.

Yours doesn't have to match this — the point is that it's true and it's
yours.

---

### Why This Matters: Future You Needs This

Picture this: you close your laptop today, and don't open this project
again for three weeks. When you come back, you won't remember the small
details — what you tried, what worked, what you were even thinking.
That's not a memory problem. That's just how time works for everyone.

The journal is what "past you" leaves for "future you" to find. Instead
of trying to reconstruct where you left off, you just read the entry
and you're back in it.

> **This is the same idea from Lab 1, now made real.** Remember the
> loop we talked about — context, build, document, re-enter? The
> journal entry you just wrote *is* the "document" step. You're not
> learning a new idea here. You're finally doing the thing we only
> talked about before.
