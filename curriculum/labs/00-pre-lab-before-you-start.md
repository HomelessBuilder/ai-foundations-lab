# Module -1 — Before You Start
## Pre-Lab: Getting Ready

*Status: Draft — for Diego's review.*

**Requirement:** This course requires a working Claude Code or Claude
Desktop account, set up and functioning, before Lab 0 begins. Creating
your Anthropic account — signing up and verifying it — is on you to
complete before you start; this pre-lab walks you through configuring
it for this course, but it does not walk you through Anthropic's account
creation process itself.

**What this covers:** the things that have to be true before Lab 0 can
even begin — a working account, and enough comfort with whichever tool
you're using to not get stuck on "where am I" or "how do I start."

**This is a standalone prerequisite, not one of the 8 core labs.** There's
no new artifact to keep here — just a short checklist of things working
before we start building anything.

---

### Before We Start

Everything from Lab 0 onward assumes two things are already true: that
your tool can actually run and bill correctly, and that you're
comfortable enough with it to follow instructions later without getting
stuck. This is where we make both of those true. It's quick, and you
only do it once.

---

### Choose Your Tool

This course works with either of Anthropic's two products — pick
whichever you already have, or whichever sounds right for you:

- **Claude Code** — terminal-based. You type instructions in a plain
  text window. The rest of this course is written with Claude Code in
  mind, so if you're comfortable with the idea of typing commands, this
  is the more direct path.
- **Claude Desktop** — a regular app window, no terminal required. If
  the idea of a terminal feels like an extra hurdle you'd rather skip,
  this is the more approachable path.

Whichever you pick, follow the matching setup path below from this point
forward.

### Part 1 — Set Up Your Anthropic Account

Both tools run on the same underlying Anthropic account, so this part is
the same either way. It needs to know who you are and how usage gets
paid for before either tool will do anything.

The account sign-up link is on this page — click it and follow the
steps. Along the way, you'll be asked to choose how you want to pay for
usage. There are generally two paths:

- **A flat-rate subscription** (billed monthly, usage included up to a
  limit)
- **Pay-as-you-go billing** (a card on file, charged based on what you
  actually use)

You don't need to deeply understand the difference to get started — if
you're not sure, the flat-rate subscription option is usually the
simpler choice for someone just getting going. Check the sign-up page
for current details; pricing and options change over time, so we won't
lock in specifics here.

> **Worth knowing:** entering billing information can feel like a bigger
> step than everything else so far — it's normal to feel a small flicker
> of hesitation there. That's a completely reasonable reaction, not a
> sign you're doing something risky. It's the same as signing up for any
> other paid tool.

If you get stuck here and can't get past it, email [SUPPORT EMAIL] and
we'll help you directly.

You'll know this part actually worked once you get to Lab 0's "Say
Hello" step and get a real response back. If something's wrong with
billing or your account, that's usually where it would show up first.

### Part 2a — Get Comfortable in a Terminal (Claude Code path)

*Skip this section if you chose Claude Desktop above.*

You'll be using a terminal throughout this course — a plain, text-only
window for typing instructions directly to your computer. On Mac it's
called Terminal. On Windows, Command Prompt, PowerShell, or Windows
Terminal. On Linux, whatever terminal app your system uses. They all
work the same basic way: type a line, press enter, something happens.

Open one now, just to see it. Nothing to build yet — just get used to
having it open.

Now practice moving it to a specific folder, since this is a skill
you'll need starting in Lab 2. Type `cd ` (with a space after it), then
drag a folder — your Desktop is fine for practice — from your regular
file browser (Finder on Mac, File Explorer on Windows, or your file
manager on Linux) directly into the terminal window. It will fill in the
folder's location for you. Press enter, and the terminal is now
"standing" in that folder.

Some systems make this even easier — right-click a folder and look for
an option like "Open Terminal Here" or "Open PowerShell window here." If
you see that, it does the same thing in one click.

> **Worth knowing:** this is just practice for now — there's no real
> task riding on it yet. You'll see this exact move again in Lab 0 and
> Lab 2, when it actually matters. Consider this your first rep, not
> your last.

### Part 2b — Get Comfortable in Claude Desktop (Claude Desktop path)

*Skip this section if you chose Claude Code above.*

Claude Desktop is a regular application window — no terminal involved.
Open it now and take a look around before we go any further. You'll want
to point it at a real folder on your computer for it to work in, the
same way Claude Code works inside whatever folder it's opened in — the
app will guide you through selecting or creating that folder and
granting it permission to read and write files there.

> **Worth knowing:** the labs in this course are written with Claude
> Code's terminal commands as the reference. If you're on Claude
> Desktop, the same ideas apply throughout — folders, files, asking it
> to save things — but you'll be doing it through the app's interface
> instead of typed commands. Where a lab says "open a terminal," read
> that as "open Claude Desktop" instead.
